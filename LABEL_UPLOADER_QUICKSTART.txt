================================================================================
USAGE SCENARIOS
================================================================================

Scenario 1: Background Monitoring
   Command: python label_uploader.py monitor
   Use when: You want to automatically upload labels as they're created
   Result: Runs continuously, uploading new PDFs automatically

Scenario 2: Initial Sync + Monitoring  
   Command: python label_uploader.py monitor --upload-existing
   Use when: First time setup or after system restart
   Result: Uploads all existing PDFs, then monitors for new ones

Scenario 3: One-Time Batch Upload
   Command: python label_uploader.py upload-all
   Use when: You just want to upload existing files once
   Result: Uploads all PDFs and exits

Scenario 4: Test Single File
   Command: python label_uploader.py test <filename.pdf>
   Use when: Testing or debugging a specific file
   Result: Uploads one file and shows detailed results

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "SFTP connection failed"
Solution: Check config.py credentials and network connectivity

Problem: "Directory does not exist"  
Solution: The script will auto-create it, or check permissions

Problem: Monitor doesn't detect new files
Solution: Ensure files are .pdf extension and in the 'label' folder

Problem: Upload fails
Solution: Run test_label_uploader.py to diagnose the issue
